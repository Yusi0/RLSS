# REDLINER 스크림 리더보드 (REDLINER Scrim Leaderboard)

REDLINER 게임의 1대1 듀얼 및 2대2 윙맨 스크림 기록을 관리하고 ELO 레이팅을 시각화하는 모던 사이버펑크 디자인의 대시보드 시스템입니다.

---

## ✨ 핵심 기능

1. **듀얼 및 윙맨 모드 완벽 분리**:
   * **1대1 듀얼 (5선승제)**: 맵 (`highrise`, `kabuki`, `arc`) / ELO 레이팅 추적
   * **2대2 윙맨 (4선승제)**: 맵 (`metro`) / ELO 레이팅 추적
   * **통합 ELO**: 두 모드의 레이팅 수치를 합산 및 평균한 통합 실력 척도 제공

2. **클릭 한 번으로 깃허브 실시간 배포 (Web Direct Push)**:
   * 웹사이트 설정창에 **GitHub Access Token**을 등록하면, 전적 입력 후 `[저장 및 GitHub 배포]` 버튼 클릭 한 번으로 `match.csv`, `lboard.csv`, `aftercalc.csv`가 깃허브 저장소로 자동 커밋 및 배포됩니다.
   * 토큰은 브라우저 로컬 저장소(`localStorage`)에만 안전하게 남으며, 공개적인 소스코드에는 전혀 노출되지 않습니다.

3. **팀 메이커 (로비 밸런서)**:
   * 리더보드 카드에서 플레이어들을 선택하여 로비에 등록한 뒤, 1v1 또는 2v2 모드를 선택하면 해당 모드의 ELO 점수를 기반으로 가장 균등한 평균 ELO를 갖는 **레드(RED) 팀**과 **블루(BLUE) 팀**을 자동 배정합니다.
   * 팀별 ELO 차이 및 예상 승률 예측(확률 %)을 시각적으로 제공합니다.

4. **플레이어 상세 분석 페이지 (`user.html`)**:
   * 개인의 1v1 및 2v2 ELO/KDR의 역대 변동 추이를 Chart.js로 미려하게 시각화합니다.
   * 4개 공식 맵별 플레이 수 및 승률 통계를 테이블로 제공합니다.
   * 개별 경기 기록 디테일과 해당 매치를 작성한 등록자(Registrant) 명을 함께 표시합니다.

---

## 📂 파일 구조

```bash
RedlinerLeaderboard/
├── index.html       # 메인 순위표, 밸런싱 로비 및 전적 입력 대시보드
├── user.html        # 플레이어 상세 전적 통계 및 추이 분석 그래프
├── elo.py           # 로컬/터미널 기반 전적 입력 및 ELO 재계산 파이썬 매니저
├── lboard.csv       # 누적 리더보드 통계 CSV
├── match.csv        # 개별 매치 로우데이터 CSV
└── aftercalc.csv    # 라운드별 ELO 및 KDR 누적 변화 기록 CSV
```

## 💻 사용 방법

### 1. 로컬에서 웹사이트 실행하기
보안 정책(CORS)으로 인해 CSV 파일을 로컬 브라우저가 읽어오려면 가벼운 HTTP 서버를 실행해야 합니다.
```bash
# RedlinerLeaderboard 디렉토리 내에서 아래 명령을 실행합니다.
npx http-server -p 8080 -c-1
```
이후 브라우저에서 `http://localhost:8080/`으로 접속하면 리더보드가 로드됩니다.

### 2. 전적 데이터 동기화 및 깃허브 연동 배포
1. 웹페이지 우측 상단 **[🔒 로그인]**을 누르고 ID와 비밀번호를 입력합니다.
2. 로그인이 성공하면 **[연동 설정]** 버튼이 생깁니다. 여기에 본인의 GitHub 계정명, 저장소 이름(Pages), 브라우저 업로드 전용 **GitHub Personal Access Token**을 등록합니다. (토큰 최소 권한 범위: `contents:write`만 필요)
3. 전적을 등록할 때 **[저장 및 GitHub 배포]**를 누르면 즉시 깃허브에 파일이 푸시되어 전 세계 모든 사용자에게 리더보드가 실시간 업데이트됩니다.

### 3. 파이썬 매니저 사용하기 (`elo.py`)
로컬 터미널 환경에서 파이썬을 이용해 직접 매치를 제어할 수도 있습니다.
```bash
python elo.py
```
* 최초 구동 시 현재 폴더에 있는 `match.csv` 데이터를 파이썬 데이터베이스(`data.db`)로 자동 마이그레이션합니다.
* 전적 등록 및 라운드 삭제를 실행하면 모든 플레이어의 ELO를 처음부터 다시 재계산하여 정밀하게 CSV 파일로 내보내 줍니다.
